Stream token usage events directly with `aiTokenStreamConsumer`, or use the [first-class Vercel AI SDK wrapper](/sdk-reference/functions/ai-token-billing/vercel-ai-sdk) that auto-instruments every LLM call.

## Stream Token Events

`aiTokenStreamConsumer` consumes an async iterable of `AITokenUsagePayload` objects and streams them to the backend for billing.

```typescript
import { scrawn } from "@scrawn/core";

const biller = scrawn({
  apiKey: process.env.SCRAWN_KEY as `scrn_${string}`,
  baseURL: process.env.SCRAWN_BASE_URL || "http://localhost:8069",
});

async function* generateUsage() {
  yield {
    userId: "user-123",
    model: "gpt-4",
    inputTokens: 100,
    outputTokens: 50,
    inputDebit: 3,     // 3¬¢ per input token
    outputDebit: 6,    // 6¬¢ per output token
  };
}

const response = await biller.aiTokenStreamConsumer(generateUsage());
// ‚Üí { eventsProcessed: 1 }
```

### Fork Mode

Return a forked stream to simultaneously bill and stream tokens to the user:

```typescript
async function* tokenStream() {
  yield {
    userId: "user-123",
    model: "gpt-4",
    inputTokens: 100,
    outputTokens: 50,
    inputDebit: 3,
    outputDebit: 6,
  };
}

const { response, stream } = await biller.aiTokenStreamConsumer(
  tokenStream(),
  { return: true }
);

// Stream tokens to user while billing runs in background
for await (const token of stream) {
  process.stdout.write(token.outputTokens.toString());
}

const result = await response;
```

## AITokenUsagePayload

`aiTokenStreamConsumer()` accepts an optional options object:

| Option | Type | Required | Default | Description |
|---|---|---|---|---|
| `return` | `boolean` | No | `false` | When `true`, returns a forked stream so you can simultaneously bill and stream tokens to the user |

### AITokenUsagePayload

Each payload in the stream represents a single token usage event:

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `userId` | `string` | Yes | ‚Äî | User to bill |
| `model` | `string` | Yes | ‚Äî | Model identifier (e.g. `"gpt-4"`) |
| `inputTokens` | `number` | Yes | ‚Äî | Input token count |
| `outputTokens` | `number` | Yes | ‚Äî | Output token count |
| `inputDebit` | `Debit` | Yes | ‚Äî | Pricing for input tokens ‚Äî number, tag, or expression |
| `outputDebit` | `Debit` | Yes | ‚Äî | Pricing for output tokens |
| `provider` | `string` | No | ‚Äî | LLM provider |
| `inputCacheTokens` | `number` | No | ‚Äî | Cached input tokens |
| `inputCacheDebit` | `Debit` | No | ‚Äî | Pricing for cached input |
| `outputCacheTokens` | `number` | No | ‚Äî | Cached output tokens |
| `outputCacheDebit` | `Debit` | No | ‚Äî | Pricing for cached output |
| `metadata` | `object` | No | ‚Äî | Arbitrary metadata |

The `Debit` type accepts a number (cents), a `biller.tag()` reference, or a pricing expression using `mul()`, `add()`, `inputTokens()`, etc.
