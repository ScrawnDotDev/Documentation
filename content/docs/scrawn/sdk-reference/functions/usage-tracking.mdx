---
title: Usage Tracking
description: Track billable events with basicUsageEventConsumer and middlewareEventConsumer
---

Events are the core of Scrawn  every API call, AI generation, or billable action produces an event. Track them manually where you need control, or automatically with middleware.

## Manual Tracking

Call `basicUsageEventConsumer` whenever a billable action happens:

```typescript
import { scrawn, tag } from "@scrawn/core";

const biller = scrawn({
  apiKey: process.env.SCRAWN_KEY as `scrn_${string}`,
  baseURL: process.env.SCRAWN_BASE_URL || "http://localhost:8069",
});

// Debit a fixed amount in cents
await biller.basicUsageEventConsumer({
  userId: "user-123",
  debit: 500,
});

// Or use a price tag managed on the backend
await biller.basicUsageEventConsumer({
  userId: "user-123",
  debit: tag("PREMIUM_CALL"),
});

// Or use a pricing expression
await biller.basicUsageEventConsumer({
  userId: "user-123",
  debit: mul(tag("PER_CALL"), 3),
});
```

### Parameters

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `userId` | `string` | Yes |  | User to bill |
| `debit` | `number \| PriceExpr` | Yes |  | Cents or pricing expression |
| `metadata` | `object` | No | `undefined` | Arbitrary metadata |

The `debit` field accepts a number (cents), a `tag()` reference, a pricing expression like `mul(tag(...), n)`, or a `biller.expr()` reference.

### Options

The second argument accepts an optional options object:

| Option | Type | Required | Default | Description |
|---|---|---|---|---|
| `onError` | `(error, context?) => void` | No | `undefined` | Callback for non-blocking error handling. `context.retry()` attempts a manual retry |

## Automatic Tracking (Middleware)

`middlewareEventConsumer` drops into any Express-compatible framework and tracks events automatically based on your extractor function.

```typescript
import { scrawn } from "@scrawn/core";

const biller = scrawn({
  apiKey: process.env.SCRAWN_KEY,
  baseURL: process.env.SCRAWN_BASE_URL,
});

app.use(biller.middlewareEventConsumer({
  extractor: (req) => ({
    userId: req.headers["x-user-id"] as string,
    debit: 1,
  }),
}));
```

### Configuration

| Option | Type | Required | Default | Description |
|---|---|---|---|---|
| `extractor` | `(req) => { userId, debit } \| null \| Promise<...>` | Yes |  | Extracts `userId` and `debit` from each request. Return `null` to skip tracking |
| `whitelist` | `string[]` | No |  | Only track specific endpoint patterns. `*` matches one segment, `**` matches multiple |
| `blacklist` | `string[]` | No |  | Exclude specific endpoints from tracking |

### Best Practices

**Place after authentication:**

```typescript
app.use(authMiddleware);
app.use(biller.middlewareEventConsumer({ /* ... */ }));
```

**Exclude payment endpoints from tracking:**

```typescript
blacklist: ["/api/collect-payment"]
```

**Middleware runs tracking in the background  your handler is not blocked:**

```typescript
app.post("/api/generate", async (req, res) => {
  const result = await generateContent(req.body);
  res.json(result);
  // Tracking already happened in parallel
});
```

### Framework Compatibility

The middleware uses standard `(req, res, next)` signatures. For Fastify, Next.js, or Hono, call `basicUsageEventConsumer` directly in route handlers or [check framework recipes](/sdk-reference/reference-and-guides/framework-recipes) for adapters.
