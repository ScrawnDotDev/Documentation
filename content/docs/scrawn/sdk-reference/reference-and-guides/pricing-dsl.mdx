title: Pricing DSL
description: Type-safe pricing expressions with mul, add, tag, inputTokens, and more
---

Scrawn includes a type-safe domain-specific language for expressing pricing logic. Use it inline wherever a `debit` or `Debit` value is accepted.

## Functions

### `tag(name)` <span className="inline-flex items-center rounded-sm border border-yellow-400 bg-yellow-400 px-2 py-0.5 font-mono text-[11px] font-black uppercase tracking-widest text-black">Events & AI</span>

Reference a price tag managed on the backend.

```typescript
import { tag } from "@scrawn/core";

tag("PREMIUM_CALL");   // Resolved to cent amount by backend
tag("EXTRA_FEE");
```

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `name` | `string` | Yes | — | The tag name registered on the backend |

### `amount(cents)` <span className="inline-flex items-center rounded-sm border border-yellow-400 bg-yellow-400 px-2 py-0.5 font-mono text-[11px] font-black uppercase tracking-widest text-black">Events & AI</span>

Explicit cent amount as an expression node (useful inside compound expressions).

```typescript
import { amount } from "@scrawn/core";

amount(500);  // $5.00
```

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `cents` | `number` | Yes | — | Amount in cents |

### `mul(...args)` <span className="inline-flex items-center rounded-sm border border-yellow-400 bg-yellow-400 px-2 py-0.5 font-mono text-[11px] font-black uppercase tracking-widest text-black">Events & AI</span>

Multiply two or more values.

```typescript
import { mul, tag } from "@scrawn/core";

mul(tag("PER_TOKEN"), 100);
mul(tag("BASE_RATE"), 2, tag("MULTIPLIER"));
```

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `...args` | `PriceExpr[]` | Yes | — | Two or more values to multiply |

### `add(...args)` <span className="inline-flex items-center rounded-sm border border-yellow-400 bg-yellow-400 px-2 py-0.5 font-mono text-[11px] font-black uppercase tracking-widest text-black">Events & AI</span>

Add two or more values.

```typescript
import { add, tag, amount } from "@scrawn/core";

add(tag("PREMIUM_CALL"), tag("EXTRA_FEE"));
add(tag("BASE"), 250, amount(100));
```

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `...args` | `PriceExpr[]` | Yes | — | Two or more values to add |

### `inputTokens()` <span className="inline-flex items-center rounded-sm border border-yellow-400 bg-yellow-400 px-2 py-0.5 font-mono text-[11px] font-black uppercase tracking-widest text-black">AI only</span>
### `outputTokens()` <span className="inline-flex items-center rounded-sm border border-yellow-400 bg-yellow-400 px-2 py-0.5 font-mono text-[11px] font-black uppercase tracking-widest text-black">AI only</span>

Token count placeholders that resolve to the actual token count of the current AI call. Only valid inside `AITokenUsagePayload` objects.

No parameters. Returns a `PriceExpr` that resolves to the token count at billing time.

```typescript
import { mul, tag, inputTokens, outputTokens } from "@scrawn/core";

mul(biller.tag("GPT_INPUT_RATE"), inputTokens());
mul(biller.tag("GPT_OUTPUT_RATE"), outputTokens());
```

## Usage in Events

Pass a pricing expression where a `debit` value is accepted:

```typescript
import { scrawn, mul, tag } from "@scrawn/core";

const biller = scrawn({
  apiKey: process.env.SCRAWN_KEY,
  baseURL: process.env.SCRAWN_BASE_URL,
});

// Tag reference
await biller.basicUsageEventConsumer({
  userId: "user-123",
  debit: tag("PREMIUM_CALL"),
});

// Compound expression
await biller.basicUsageEventConsumer({
  userId: "user-123",
  debit: mul(tag("PER_CALL"), 3),
});
```

For AI token billing, use `inputTokens()` and `outputTokens()` as placeholders:

```typescript
async function* generateUsage() {
  yield {
    userId: "user-123",
    model: "gpt-4",
    inputTokens: 100,
    outputTokens: 50,
    inputDebit: mul(biller.tag("GPT_INPUT_RATE"), inputTokens()),
    outputDebit: mul(biller.tag("GPT_OUTPUT_RATE"), outputTokens()),
  };
}

await biller.aiTokenStreamConsumer(generateUsage());
```

## Persisted Expressions

Store complex expressions on the backend and reference them by name:

```typescript
// Reference a saved expression
await biller.basicUsageEventConsumer({
  userId: "user-123",
  debit: biller.expr("MY_PRICING_EXPR"),
});
```
## Error Handling

Invalid expressions (wrong types, missing operands, etc.) throw `PricingExpressionError` at runtime. The TypeScript types catch most issues at compile time.
