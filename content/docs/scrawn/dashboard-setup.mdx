The Scrawn Dashboard is a Next.js application that gives you a visual interface to manage API keys, track events, and monitor your billing infrastructure.

![Scrawn Dashboard](/Dashboard.png)

## Prerequisites

Before setting up the dashboard, make sure you have:
- [Scrawn backend running](/local-setup) locally
- Database URL (same PostgreSQL database as your backend)
- Initial API key generated from [Quick Start](/quick-start#5-generate-initial-api-key)

  <Tab value="Windows (PowerShell)">
    ```powershell

    ```

Edit `.env.local` with your configuration:

```bash title=".env.local"
DATABASE_URL=postgresql://user:password@localhost:5432/scrawn
SCRAWN_BASE_URL=http://localhost:8069
SCRAWN_KEY=scrn_YOUR_SCRAWN_KEY_HERE
```

<Callout type="info" title="Configuration Details">
**DATABASE_URL:** Use a read-only database user for security (same database as your backend)

**SCRAWN_BASE_URL:** Your deployed Scrawn backend URL or `http://localhost:8069` for local testing

**SCRAWN_KEY:** The initial API key you generated in the [Quick Start guide](/quick-start#5-generate-initial-api-key)
</Callout>

## 5. Dashboard Onboarding

When you first open the dashboard, you will enter an onboarding flow that requires your Dodo Payments API keys and Product IDs.

### 5.1. Dodo API Keys

To get your Dodo Live API Key and Dodo Test API Key:
1. Go to the Dodo Payments dashboard.
2. Navigate to **Developer > API Keys** in the sidebar.
3. While in **Test Mode**, click **Add API Key** to generate your Test API key.
4. Click the **Live Mode** toggle in the bottom left corner to switch environments.
5. Generate your Live API key in the same way.

![Dodo Mode Toggle](/Dodo%20Live%20Test.jpeg)

Copy and paste both of these keys into the onboarding flow in your Scrawn Dashboard.

### 5.2. Dodo Products

Next, you need to create a product in Dodo to collect payments. You will need to create two products: one in **Test Mode** and one in **Live Mode**.

1. In the Dodo Payments dashboard, navigate to **Products > Product** in the sidebar.
2. Click to **Add New Product**.
3. Fill in the Basic Details (Product Name, Brand, Tax Category, etc.). You can set these to whatever makes sense for your application.

![Dodo Add Product](/DODO%20PRODUCT%201.png)

4. Scroll down to the **Pricing** section.
5. **CRITICAL STEP:** Ensure you select **One Time** pricing, and check the box for **Pay What You Want**. Set the **Minimum Price** to `$0`. (This is the only configuration value Scrawn strictly requires; the rest can be whatever you want).

![Dodo Product Pricing Configuration](/DODO%20PRODUCT%202.png)

6. Save the product and copy its **Product ID**. 
7. Switch to the other environment mode (Test or Live) using the toggle in the bottom left, create the second product with the exact same Pay What You Want configuration, and copy its Product ID as well.

Paste both the **Test Product ID** and **Live Product ID** into the respective fields in your Scrawn Dashboard onboarding flow to complete your setup.

## Creating a New API Key

Once your dashboard is running, you can easily generate new API keys:

### 1. Navigate to API Keys Page

Open your dashboard and click on **API Keys** in the sidebar, or navigate to `/dashboard/api-keys`.

### 2. Click Create API Key

Click the **Create API Key** button in the top-right corner.

### 3. Fill in Details

A dialog will appear asking for:
- **API Key Name:** Give it a descriptive name (e.g., "Production API", "Development Key")
- **Expires In (days):** How many days until the key expires (default: 365)

### 4. Copy Your Key

<Callout type="warn" title="Important">
After clicking **Create API Key**, you'll see the generated key **only once**. Make sure to copy it immediately!
</Callout>

Click the copy button next to the key to copy it to your clipboard.

### 5. Use Your New Key

You can now use this API key in your application:

```typescript
import { scrawn } from "@scrawn/core";

const biller = scrawn({
  apiKey: "scrn_YOUR_NEW_KEY_HERE" as `scrn_${string}`, // [!code highlight]
  baseURL: process.env.SCRAWN_BASE_URL,
});
```

## Dashboard Features

The dashboard provides several views to monitor your billing infrastructure:

- **Overview:** Real-time revenue, API usage, and trends
- **API Keys:** Create, view, and manage all API keys
- **Users:** Track user balances and activity
- **Events:** Monitor SDK call events and costs
